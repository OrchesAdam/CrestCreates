# CrestCreates 代码生成方案优化建议

## 概述

本文档针对 CrestCreates 框架当前代码生成方案存在的问题，提出一套更灵活、更协调的优化方案。

---

## 当前问题分析

### 1. 生成器碎片化
- **现状**：多个独立源生成器各自为政
  - `RepositorySourceGenerator`
  - `CrudServiceSourceGenerator`
  - `QueryBuilderSourceGenerator`
  - `ControllerSourceGenerator`
  - 等等

- **问题**：缺乏统一协调，生成的代码可能冲突或重叠

### 2. 过度生成，灵活性丧失
- **现状**：加特性就生成一切，无法选择性生成
- **问题**：
  - 无法只生成部分功能
  - 难以在自动生成和自定义之间取得平衡
  - 容易与手写代码冲突

### 3. 分层和职责不清
- **现状**：Repository、Service、QueryBuilder 功能重叠
- **问题**：
  - Repository 已有查询方法，QueryBuilder 又生成一套
  - CRUD Service 再包一层，层次过多
  - 不知道业务逻辑应该放在哪一层

### 4. 扩展性不足
- **现状**：缺乏钩子和扩展点
- **问题**：
  - 无法注入自定义逻辑
  - 生成代码与手写代码难以协同
  - partial 机制使用不够恰当

### 5. 中间层过于复杂
- **现状**：过度依赖 `FilterDescriptor`、`SortDescriptor` 等字符串描述符
- **问题**：
  - 不够类型安全
  - 增加了不必要的复杂度
  - 性能可能有损耗

---

## 优化方案设计

### 核心原则

1. **统一协调** - 一个主生成器，统一管理所有生成逻辑
2. **渐进式生成** - 可以选择性生成，部分自定义
3. **清晰分层** - 每一层有明确职责，不重叠
4. **可扩展** - 提供钩子和扩展点
5. **简单** - 减少中间层，保持简洁

---

### 方案一：统一生成器 + 特性配置

#### 1. 统一特性设计

```csharp
/// <summary>
/// 统一的实体配置特性
/// </summary>
[AttributeUsage(AttributeTargets.Class, AllowMultiple = false)]
public class GenerateEntityAttribute : Attribute
{
    /// <summary>
    /// 是否生成仓储
    /// </summary>
    public bool GenerateRepository { get; set; } = true;
    
    /// <summary>
    /// 是否生成 CRUD 服务
    /// </summary>
    public bool GenerateCrudService { get; set; } = true;
    
    /// <summary>
    /// 是否生成查询扩展方法
    /// </summary>
    public bool GenerateQueryExtensions { get; set; } = true;
    
    /// <summary>
    /// 是否生成控制器
    /// </summary>
    public bool GenerateController { get; set; } = false;
    
    /// <summary>
    /// 控制器路由
    /// </summary>
    public string? ControllerRoute { get; set; }
    
    /// <summary>
    /// ORM 提供者
    /// </summary>
    public OrmProvider OrmProvider { get; set; } = OrmProvider.EfCore;
    
    /// <summary>
    /// 需要排除的属性（用于 DTO 生成）
    /// </summary>
    public string[]? ExcludeProperties { get; set; }
    
    /// <summary>
    /// 生成基类而非完整实现（允许继承扩展）
    /// </summary>
    public bool GenerateAsBaseClass { get; set; } = false;
}
```

#### 2. 统一生成器

创建一个 `UnifiedSourceGenerator`，统一管理所有生成逻辑：

```csharp
[Generator]
public class UnifiedSourceGenerator : IIncrementalGenerator
{
    public void Initialize(IncrementalGeneratorInitializationContext context)
    {
        // 统一处理所有带 [GenerateEntity] 的实体
        var entities = context.SyntaxProvider
            .CreateSyntaxProvider(
                predicate: IsEntityCandidate,
                transform: GetEntityInfo)
            .Where(x => x != null)
            .Collect();

        context.RegisterSourceOutput(entities, ExecuteGeneration);
    }

    private void ExecuteGeneration(
        SourceProductionContext context,
        ImmutableArray<EntityInfo?> entities)
    {
        foreach (var entity in entities)
        {
            if (entity == null) continue;

            // 根据配置选择性生成
            if (entity.Config.GenerateRepository)
            {
                GenerateRepository(context, entity);
            }

            if (entity.Config.GenerateQueryExtensions)
            {
                GenerateQueryExtensions(context, entity);
            }

            if (entity.Config.GenerateCrudService)
            {
                GenerateCrudService(context, entity);
            }

            if (entity.Config.GenerateController)
            {
                GenerateController(context, entity);
            }
        }
    }
}
```

### 方案二：基类 + 继承模式（推荐）

#### 设计理念

生成**基类**，让开发者继承并扩展，而不是直接生成完整实现。

#### 示例：仓储层

**生成的基类**：
```csharp
// <auto-generated />
public abstract class BookRepositoryBase : EfCoreRepository<Book, Guid>, IBookRepository
{
    protected BookRepositoryBase(DbContext dbContext) : base(dbContext) { }

    // 生成的基础 CRUD 方法
    public virtual Task<Book?> GetByIdAsync(Guid id, CancellationToken ct = default)
    {
        return DbContext.Set<Book>().FindAsync(new object[] { id }, ct).AsTask();
    }

    // 更多基础方法...
}
```

**开发者手写的实现**：
```csharp
public partial class BookRepository : BookRepositoryBase
{
    public BookRepository(DbContext dbContext) : base(dbContext) { }

    // 可以重写基类方法
    public override Task<Book?> GetByIdAsync(Guid id, CancellationToken ct = default)
    {
        // 自定义逻辑，比如添加 Include
        return DbContext.Set<Book>()
            .Include(b => b.Category)
            .FirstOrDefaultAsync(b => b.Id == id, ct);
    }

    // 可以添加自定义方法
    public Task<Book?> GetByIsbnAsync(string isbn, CancellationToken ct = default)
    {
        return DbContext.Set<Book>()
            .FirstOrDefaultAsync(b => b.ISBN == isbn, ct);
    }
}
```

#### 示例：查询扩展（替代 QueryBuilder）

**生成的查询扩展方法**：
```csharp
// <auto-generated />
public static class BookQueryExtensions
{
    public static IQueryable<Book> WhereTitle(this IQueryable<Book> query, string title)
    {
        return query.Where(b => b.Title == title);
    }

    public static IQueryable<Book> WhereTitleContains(this IQueryable<Book> query, string keyword)
    {
        return query.Where(b => b.Title.Contains(keyword));
    }

    public static IQueryable<Book> WhereAuthor(this IQueryable<Book> query, string author)
    {
        return query.Where(b => b.Author == author);
    }

    public static IQueryable<Book> WhereStatus(this IQueryable<Book> query, BookStatus status)
    {
        return query.Where(b => b.Status == status);
    }

    // 排序扩展
    public static IQueryable<Book> OrderByTitle(this IQueryable<Book> query, bool ascending = true)
    {
        return ascending ? query.OrderBy(b => b.Title) : query.OrderByDescending(b => b.Title);
    }

    // 更多扩展方法...
}
```

**使用方式**：
```csharp
// 直接链式调用，无需中间层
var books = await dbContext.Set<Book>()
    .WhereTitleContains("C#")
    .WhereAuthor("Martin")
    .WhereStatus(BookStatus.Available)
    .OrderByTitle()
    .ToListAsync();
```

### 方案三：服务层设计

#### 生成的服务基类

```csharp
// <auto-generated />
public abstract class BookCrudServiceBase : IBookCrudService
{
    protected readonly IBookRepository _repository;
    protected readonly IMapper _mapper;

    protected BookCrudServiceBase(IBookRepository repository, IMapper mapper)
    {
        _repository = repository;
        _mapper = mapper;
    }

    public virtual async Task<BookDto> CreateAsync(CreateBookDto input, CancellationToken ct = default)
    {
        var entity = _mapper.Map<Book>(input);
        entity = await _repository.AddAsync(entity, ct);
        return _mapper.Map<BookDto>(entity);
    }

    // 钩子方法：创建前
    protected virtual Task OnCreatingAsync(Book entity, CancellationToken ct = default)
        => Task.CompletedTask;

    // 钩子方法：创建后
    protected virtual Task OnCreatedAsync(Book entity, CancellationToken ct = default)
        => Task.CompletedTask;

    // 更多方法...
}
```

#### 开发者手写的服务实现

```csharp
public class BookAppService : BookCrudServiceBase, IBookAppService
{
    private readonly ICategoryRepository _categoryRepository;

    public BookAppService(
        IBookRepository repository,
        IMapper mapper,
        ICategoryRepository categoryRepository)
        : base(repository, mapper)
    {
        _categoryRepository = categoryRepository;
    }

    // 重写 CreateAsync，添加业务逻辑
    public override async Task<BookDto> CreateAsync(CreateBookDto input, CancellationToken ct = default)
    {
        // 自定义验证
        if (!await _repository.IsIsbnUniqueAsync(input.ISBN, ct))
            throw new Exception($"ISBN '{input.ISBN}' already exists");

        // 调用基类方法
        return await base.CreateAsync(input, ct);
    }

    // 钩子方法：创建前
    protected override async Task OnCreatingAsync(Book entity, CancellationToken ct = default)
    {
        // 可以在这里添加创建前的逻辑
        await base.OnCreatingAsync(entity, ct);
    }

    // 添加自定义方法
    public async Task<BookDto?> GetByIsbnAsync(string isbn, CancellationToken ct = default)
    {
        var book = await _repository.GetByIsbnAsync(isbn, ct);
        return book != null ? _mapper.Map<BookDto>(book) : null;
    }
}
```

## 推荐的完整方案

结合以上方案，推荐采用以下设计：

### 1. 统一特性

```csharp
[GenerateEntity(
    GenerateRepository = true,
    GenerateQueryExtensions = true,
    GenerateCrudService = true,
    GenerateController = false,
    OrmProvider = OrmProvider.EfCore,
    GenerateAsBaseClass = true)]
public class Book : AuditedEntity<Guid>
{
    // ...
}
```

### 2. 生成内容

生成以下文件：
- `BookRepositoryBase.g.cs` - 仓储基类
- `IBookRepository.g.cs` - 仓储接口（partial）
- `BookQueryExtensions.g.cs` - 查询扩展方法
- `BookCrudServiceBase.g.cs` - CRUD 服务基类
- `IBookCrudService.g.cs` - CRUD 服务接口（partial）
- `BookDto.g.cs`、`CreateBookDto.g.cs`、`UpdateBookDto.g.cs` - DTO
- `BookMappingProfile.g.cs` - AutoMapper 配置

### 3. 开发者需要手写的内容

- `BookRepository.cs` - 继承基类，添加自定义逻辑
- `BookAppService.cs` - 继承基类，添加业务逻辑
- 控制器（可选）

## 优势总结

### 1. 灵活性
- 可以选择性生成部分功能
- 基类模式允许完全重写任何方法
- 查询扩展方法直接操作 `IQueryable`，非常灵活

### 2. 可维护性
- 生成代码和手写代码分离清晰
- partial 机制让接口可以扩展
- 钩子方法提供明确的扩展点

### 3. 性能
- 减少中间层，直接操作表达式树
- 查询扩展方法编译时确定，无运行时开销

### 4. 开发体验
- 统一的特性配置
- 清晰的分层
- 类型安全的查询

---

## 迁移建议

### 阶段一：统一生成器
1. 创建 `UnifiedSourceGenerator`
2. 保留现有生成器作为过渡
3. 逐步迁移到统一生成器

### 阶段二：基类模式
1. 修改生成器，默认生成基类
2. 保持向后兼容（通过特性控制）
3. 文档说明迁移步骤

### 阶段三：简化查询层
1. 移除 `FilterDescriptor`、`SortDescriptor`
2. 生成查询扩展方法替代
3. 更新文档和示例

---

## 总结

这个优化方案的核心思想是：
1. **统一** - 一个生成器管所有
2. **基类** - 生成基类，让你继承
3. **简单** - 减少中间层，保持直接
4. **灵活** - 提供多个扩展点

这样既保持了代码生成的便利性，又不会丧失灵活性。
