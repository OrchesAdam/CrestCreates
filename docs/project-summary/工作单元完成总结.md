## 🎯 核心特性

### 1. **多 ORM 支持**
```csharp
// EF Core
var efUow = _factory.Create(OrmProvider.EfCore);

// SqlSugar
var sqlSugarUow = _factory.Create(OrmProvider.SqlSugar);

// FreeSql
var freeSqlUow = _factory.Create(OrmProvider.FreeSql);
```

### 2. **自动审计**
```csharp
var product = new Product(Guid.NewGuid(), "Test", 99.99m);
await _repository.AddAsync(product);
await _uow.SaveChangesAsync();

// product.CreationTime - 自动填充
// product.CreatorId - 自动填充
```

### 3. **软删除**
```csharp
await _repository.DeleteAsync(product);
await _uow.SaveChangesAsync();

// 转换为软删除：
// product.IsDeleted = true
// product.DeletionTime = DateTime.UtcNow
// product.DeleterId = currentUserId
```

### 4. **简化的事务管理**
```csharp
return await _uowManager.ExecuteAsync(async uow =>
{
    // 自动 BeginTransaction
    var order = new Order();
    await _repository.AddAsync(order);
    await uow.SaveChangesAsync();
    // 自动 Commit
    return order;
    // 异常时自动 Rollback
});
```

## 🏗️ 架构设计

### 分层架构
```
Application Layer (Service)
        ↓
IUnitOfWorkManager / IUnitOfWork
        ↓
┌───────────┬───────────┬───────────┐
│  EfCore   │ SqlSugar  │  FreeSql  │
│   UoW     │   UoW     │   UoW     │
└───────────┴───────────┴───────────┘
        ↓
┌───────────────────────────────────┐
│      Audit Interceptors           │
│  (自动填充审计字段/软删除处理)      │
└───────────────────────────────────┘
```

### 关键接口
```csharp
public interface IUnitOfWork : IDisposable
{
    Task BeginTransactionAsync();
    Task CommitTransactionAsync();
    Task RollbackTransactionAsync();
    Task<int> SaveChangesAsync();
}

public interface IUnitOfWorkFactory
{
    IUnitOfWork Create(OrmProvider provider);
}

public interface IUnitOfWorkManager
{
    IUnitOfWork Current { get; }
    IUnitOfWork Begin(OrmProvider? provider = null);
    Task<TResult> ExecuteAsync<TResult>(...);
}

public interface ICurrentUserProvider
{
    Guid? GetCurrentUserId();
}
```

## 💡 技术亮点

### 1. **统一的 API**
三种 ORM 使用相同的接口，便于切换和测试。

### 2. **自动化审计**
无需手动填充审计字段，拦截器自动处理。

### 3. **软删除透明化**
查询自动过滤已删除数据，物理删除自动转换为软删除。

### 4. **异常安全**
使用 `using` 模式和 `try-finally` 确保事务正确回滚。

### 5. **依赖注入友好**
完全基于依赖注入，易于测试和扩展。

## 📝 使用注意事项

### 1. **必须实现 ICurrentUserProvider**
```csharp
// ❌ 不要使用默认实现
services.AddScoped<ICurrentUserProvider, DefaultCurrentUserProvider>();

// ✅ 实现真实的用户获取逻辑
services.AddScoped<ICurrentUserProvider, HttpContextCurrentUserProvider>();
```

### 2. **注册所有需要的服务**
```csharp
// 注册工作单元
services.AddUnitOfWork(OrmProvider.EfCore);

// 注册具体实现
services.AddScoped<EfCoreUnitOfWork>();
services.AddScoped<SqlSugarUnitOfWork>();
services.AddScoped<FreeSqlUnitOfWork>();

// 注册拦截器依赖
services.AddScoped<ICurrentUserProvider, YourImplementation>();
```

### 3. **配置审计拦截器**
```csharp
// EF Core
options.AddInterceptors(new AuditInterceptor(currentUserProvider));

// SqlSugar
sqlSugarClient.ConfigureAuditInterceptor(currentUserProvider);

// FreeSql
freeSql.ConfigureAuditInterceptor(currentUserProvider);
```
