# CrestCreates 框架 - 产品需求文档

## 概述
- **Summary**: CrestCreates 是一个基于 ASP.NET Core 和源代码生成器的模块化框架，专注于领域驱动设计 (DDD) 和基础设施即代码 (IaC) 能力，支持多租户、ORM 抽象、代码生成等核心功能。
- **Purpose**: 提供一个高性能、模块化的框架，对标 Abp Framework，通过 C# SourceGenerator 减少反射和程序集扫描，最终实现本地 AoT 运行。
- **Target Users**: 开发团队、架构师、企业应用开发者，需要构建模块化、高性能的企业级应用。

## 目标
- 构建模块化架构，支持功能按需引入和独立部署
- 提供统一的 ORM 抽象，支持多种 ORM 框架
- 实现完整的多租户解决方案
- 开发自动化代码生成工具（SourceGenerator）
- 支持完整的领域驱动设计实践
- 提供完整的云原生支持
- 减少反射和程序集扫描，优化性能
- 最终实现本地 AoT 运行

## 非目标
- 不支持非 .NET 平台
- 不提供前端框架集成
- 不包含业务逻辑实现
- 不替代现有的 ORM 框架，而是提供抽象层

## 背景与上下文
- 现有框架如 Abp Framework 依赖大量反射和程序集扫描，影响性能
- .NET 6+ 提供了更好的 SourceGenerator 支持，可用于减少运行时反射
- 云原生应用需要更模块化、可扩展的架构
- 领域驱动设计在复杂企业应用中越来越重要

## 功能需求
- **FR-1**: 模块化架构
  - 支持模块的注册、发现和依赖管理
  - 支持模块的独立开发和部署
  - 提供模块生命周期管理

- **FR-2**: ORM 抽象
  - 提供统一的数据访问接口
  - 支持 Entity Framework Core、FreeSql、SqlSugar 等 ORM 框架
  - 提供仓储模式实现

- **FR-3**: 多租户支持
  - 提供租户解析和管理
  - 支持不同的租户隔离策略
  - 提供租户相关的基础设施

- **FR-4**: 代码生成
  - 基于 SourceGenerator 的自动化代码生成
  - 支持实体、服务、控制器等代码生成
  - 减少手动代码编写

- **FR-5**: DDD 支持
  - 提供实体、值对象、聚合根等 DDD 概念的实现
  - 支持领域事件和领域服务
  - 提供仓储模式的实现

- **FR-6**: 云原生支持
  - 支持容器化部署
  - 提供健康检查、监控等云原生功能
  - 支持分布式事务和事件总线

- **FR-7**: 安全与授权
  - 提供权限管理和认证功能
  - 支持基于角色的访问控制
  - 提供安全相关的基础设施

- **FR-8**: 缓存系统
  - 提供多级缓存支持
  - 支持内存缓存和 Redis 缓存
  - 提供缓存键生成和管理

- **FR-9**: 事件总线
  - 提供本地和分布式事件总线
  - 支持 Kafka、RabbitMQ 等消息中间件
  - 提供事件存储和重试机制

- **FR-10**: 健康检查
  - 提供应用健康状态监控
  - 支持自定义健康检查
  - 提供健康检查 API

## 非功能需求
- **NFR-1**: 性能优化
  - 减少反射和程序集扫描
  - 优化内存使用
  - 支持本地 AoT 编译

- **NFR-2**: 可扩展性
  - 模块化设计，支持插件式扩展
  - 提供统一的扩展点
  - 支持自定义模块和组件

- **NFR-3**: 可维护性
  - 清晰的代码结构和文档
  - 遵循最佳实践和设计模式
  - 提供详细的开发指南

- **NFR-4**: 可靠性
  - 提供事务管理和错误处理
  - 支持分布式事务
  - 提供事件重试机制

## 约束
- **技术**: .NET 6+、ASP.NET Core、C# SourceGenerator
- **业务**: 无特定业务约束
- **依赖**: 第三方库如 Entity Framework Core、Redis、Kafka、RabbitMQ 等

## 假设
- 开发团队熟悉 .NET 生态系统和 DDD 概念
- 项目将在支持 .NET 6+ 的环境中运行
- 目标是构建企业级应用框架

## 验收标准

### AC-1: 模块化架构
- **Given**: 一个包含多个模块的应用
- **When**: 启动应用时
- **Then**: 所有模块正确注册和初始化
- **Verification**: `programmatic`

### AC-2: ORM 抽象
- **Given**: 配置了不同的 ORM 提供商
- **When**: 使用仓储模式访问数据
- **Then**: 代码无需修改即可切换 ORM 框架
- **Verification**: `programmatic`

### AC-3: 多租户支持
- **Given**: 一个多租户应用
- **When**: 不同租户访问应用
- **Then**: 租户数据正确隔离
- **Verification**: `programmatic`

### AC-4: 代码生成
- **Given**: 定义了实体和服务
- **When**: 编译项目时
- **Then**: 自动生成相关代码
- **Verification**: `programmatic`

### AC-5: DDD 支持
- **Given**: 使用 DDD 概念构建应用
- **When**: 实现领域逻辑
- **Then**: 框架提供必要的基础设施
- **Verification**: `human-judgment`

### AC-6: 云原生支持
- **Given**: 部署到云环境
- **When**: 运行应用
- **Then**: 应用正常运行并提供云原生功能
- **Verification**: `programmatic`

### AC-7: 性能优化
- **Given**: 高负载场景
- **When**: 运行应用
- **Then**: 应用性能符合预期
- **Verification**: `programmatic`

### AC-8: 可扩展性
- **Given**: 需要扩展框架功能
- **When**: 添加自定义模块
- **Then**: 模块正确集成到框架中
- **Verification**: `programmatic`
